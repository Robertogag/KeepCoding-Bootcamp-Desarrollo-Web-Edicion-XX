## Module Overview

This module focused on the foundations of relational database design and SQL, moving from a business requirement to a structured and normalized PostgreSQL solution.

The work completed in this module combined conceptual modeling with implementation, allowing the transition from entity-relationship design to executable SQL scripts.

---

## Main Topics Covered

- Entity-relationship modeling
- Entities, attributes, relationships and cardinality
- Primary keys and foreign keys
- One-to-one, one-to-many and many-to-many relationships
- Relational model design from business requirements
- Normalization
- First, second and third normal form
- SQL fundamentals in PostgreSQL
- DDL: schema creation, table creation, alteration and deletion
- DML: inserts, selects, updates, deletes and joins

---

## What Was Learned

Throughout this module, the main objective was to understand how to transform a real business scenario into a well-structured relational database model.

This included:

- identifying the core entities of a system
- defining the attributes for each entity
- establishing correct relationships between tables
- deciding primary keys and foreign keys
- reducing redundancy through normalization
- organizing information in a clear and scalable model
- implementing the design in PostgreSQL using SQL
- validating the model with inserts and functional queries

---

## Module Goal

The goal of this module was to learn how to move from a business problem to a normalized relational model and then implement that solution in SQL.
