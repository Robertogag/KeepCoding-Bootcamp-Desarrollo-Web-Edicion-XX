# SQL Modeling

![Module image](00_images/sql.jpg)

## Module summary

This module focused on the foundations of relational database design and SQL, from understanding a business requirement to implementing a normalized solution in PostgreSQL.

During the module, the main topics covered were:

- entity-relationship modeling
- entities, attributes, relationships and cardinality
- primary keys and foreign keys
- one-to-one, one-to-many and many-to-many relationships
- normalization
- first, second and third normal form
- relational database design from business requirements
- SQL basics in PostgreSQL
- DDL: schemas, table creation, table modification and deletion
- DML: inserts, selects, updates, deletes and joins

## What was learned in this module

Throughout this module, the main learning objective was to understand how to move from a real business scenario to a well-structured relational database model.

This included:

- identifying the main entities of a problem
- defining attributes for each entity
- establishing correct relationships between tables
- deciding primary and foreign keys
- reducing redundancy through normalization
- designing a relational model that is clear, consistent and scalable
- implementing that model using SQL scripts in PostgreSQL
- validating the database using inserts and functional queries

## Goal of the module

The objective of this module was to learn how to transform a business problem into a normalized relational model and then implement that model in SQL.
